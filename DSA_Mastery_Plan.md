## ⚙️ Learning Strategy

### 🧩 Core Principle
Focus on **patterns, not problems**.

Every problem must map to a known pattern:
- Arrays & Hashing
- Sliding Window
- Two Pointers
- Stack / Monotonic Stack
- Binary Search
- Recursion & Backtracking
- Trees & BST
- Heaps (Priority Queue)
- Graphs (BFS/DFS/Topo)
- Dynamic Programming

---

## ⏱️ Daily 1-Hour Plan

### ⏳ 0–5 min → Recap
- What pattern did you learn yesterday?
- What mistake did you make?

### 🧠 5–15 min → Pattern Understanding
- Learn 1 pattern deeply:
  - When to use it
  - Key signals in questions
  - Time/space tradeoffs

### 💻 15–50 min → Problem Solving
- Solve **1–2 curated problems**
- Focus:
  - Pattern recognition
  - Clean Java implementation
  - Avoid brute force

### 🔍 50–60 min → Review & Document
- Analyze:
  - What went wrong?
  - Could it be optimized?
  - Did you miss edge cases?
- **Document (notes.md):** 
  - **Brute Force:** Logic + Complexity.
  - **Optimal Solution:** Pattern-based logic + Complexity.
  - **Detailed Dry Run:** Step-by-step trace of the optimal logic.

---

## 🧠 Problem-Solving Framework

Before coding, ALWAYS answer:

1. What pattern fits this problem?
2. Can I reduce time complexity?
3. What data structure helps here?
4. What are edge cases?

---

## 📦 Coding Standards (Java)

- Use meaningful variable names
- Prefer `HashMap`, `HashSet`, `ArrayList`
- Avoid unnecessary nested loops
- Write modular code (helper methods if needed)
- Handle edge cases explicitly

---

## 🚦 Edge Case Checklist

Always test for:
- Empty input
- Single element
- Large input size
- Duplicate values
- Negative numbers
- Sorted / reverse sorted input

---

## ⏱️ Complexity Targets

| Pattern            | Expected Time |
|--------------------|--------------|
| Arrays/Hashing     | O(N)         |
| Sliding Window     | O(N)         |
| Binary Search      | O(log N)     |
| Trees/Graphs       | O(N + E)     |
| DP                 | O(N) / O(N²) |
