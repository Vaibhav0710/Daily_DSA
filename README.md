# 🎯 DSA Mastery: The 90-Day FAANG Plan

**Goal**: Crack Google/FAANG-level SDE Roles  
**Timeline**: Late 2026 / Early 2027  
**Constraint**: 1 Hour / Day  
**Language**: Java  
**Core Rule**: **Patterns, not problems.** 

---

## ⏱️ Daily 1-Hour Session (The Routine)
- **0–5 min (Recap):** Recall yesterday's pattern and mistakes.
- **5–15 min (Learn):** Deep dive into 1 pattern (signals, time/space tradeoffs).
- **15–50 min (Solve):** Code 1–2 curated problems. Focus on clean Java implementation.
- **50–60 min (Review):** Debug failures, optimize complexity, test edge cases (empty inputs, single elements, negative numbers).

## 🚨 Strict Coding Framework (Before writing code)
1. What **pattern** fits this problem?
2. Can I achieve optimal **Time/Space complexity**?
3. What data structure solves the bottleneck? (e.g. `HashMap`, `PriorityQueue`).
4. What are the tricky **edge cases**?

### Expected Complexity Targets:
- Arrays/Hashing/Sliding Window: `O(N)`
- Binary Search: `O(log N)`
- Trees/Graphs: `O(N + E)`
